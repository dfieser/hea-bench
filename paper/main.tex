\title{%
  hea-bench: an open benchmark suite and reference baselines for
  high-entropy alloy phase prediction
}

\author[1]{David Fieser\corref{cor1}}

\begin{abstract}
Empirical phase-prediction rules for high-entropy alloys are widely
used as fast pre-screens for candidate compositions. The four most
common rules are the entropy criterion of \citet{yeh2004}, the
atomic-size mismatch criterion of \citet{zhang2008}, the
valence-electron-concentration criterion of \citet{guo2011}, and
the \Omegaparam{}-parameter criterion of \citet{yang2012}. Their
quoted accuracies, however, come from heterogeneous and mutually
incompatible datasets, and no published artifact treats them as
proper diagnostic classifiers on a common open benchmark. This work
presents \hea{}, a Python library and reproducible benchmark suite
that consolidates three primary open phase-label datasets into a
single \num{7784}-composition resource with explicit per-row source
provenance and cross-source label-conflict tracking, implements all
four canonical rules as binary or multi-class diagnostic classifiers
with full statistics including accuracy, sensitivity, specificity,
Wilson 95\% intervals, receiver-operating-characteristic curves, and
Youden's $J$, and ships in two interchangeable user surfaces. The
first surface is a standard installable Python package. The second
is an in-browser frontend that runs the same wheel via Pyodide on
WebAssembly, requiring no local installation. On the consolidated
benchmark, both binary rules collapse to ``predict single-phase
almost always'' with Youden's $J$ values of 0.075 and 0.032.
A receiver-operating-characteristic sweep of the \citet{zhang2008}
$\delta$ threshold finds an optimum at $\delta < \SI{2.5}{\percent}$
rather than the canonical $\delta < \SI{6.5}{\percent}$. The package
is released as \hea{} 0.0.1 with 151 tests, including pinned
regression tests on every reported benchmark number, under the MIT
License.
\end{abstract}

\section{Motivation and significance}
% =====================================================================

High-entropy alloys, introduced independently by
\citet{cantor2004} and \citet{yeh2004} in 2004, are multi-component
alloys with five or more principal elements near equiatomic
composition. The configurational mixing entropy in such alloys is
large enough to stabilize disordered solid solutions over the
intermetallic compounds predicted by classical Hume-Rothery rules.
The compositional space is vast, combinatorially larger than that
of conventional alloys, which makes \emph{pre-screening} a
practical necessity before any expensive density-functional-theory,
CALPHAD, or experimental campaign.

Four empirical phase-prediction rules dominate the high-entropy
alloy literature. The first is the configurational mixing entropy
criterion of \citet{yeh2004}, which classifies an alloy as
high-entropy when $\Smix > 1.5 R$. The second is the atomic-size
mismatch criterion of \citet{zhang2008}, which predicts a
single-phase solid solution when $\delta < \SI{6.5}{\percent}$. The
third is the valence-electron-concentration criterion of
\citet{guo2011}, which predicts an FCC structure at $\mathrm{VEC}
\geq 8.0$ and a BCC structure at $\mathrm{VEC} < 6.87$. The fourth
is the $\Omegaparam$-parameter criterion of \citet{yang2012}, with
$\Omegaparam = \Tm \, \Smix \, / \, |\Hmix|$ and a threshold of
$\Omegaparam > 1.1$ for single-phase stability. Each rule is
supported in its original publication by classification rates of
roughly \SI{80}{\percent} on the dataset available at the time.

Three gaps in the open literature motivate this work.
First, every published evaluation of these rules uses a different
dataset, with different preprocessing, deduplication, and phase
labeling. Head-to-head comparison is therefore impossible. To our
knowledge there is no published, open, \citet{dunn2020matbench}-style
benchmark suite for high-entropy alloy phase classification.
Second, the rules are typically reported as accuracies of the form
$X / N$, without the diagnostic-classifier machinery, namely
sensitivity, specificity, Wilson intervals, receiver-operating-
characteristic analysis, and Youden's $J$, that would expose where
they actually generalize.
Third, the only widely used interactive tool for Miedema-style
enthalpy calculations is the Windows-only, registration-gated
\textsc{MiedCalc} \citep{zhang2016miedema}, and no zero-install
open analogue exists.

\hea{} addresses all three. We consolidate three primary open
phase-label datasets into a single \num{7784}-composition resource
with full provenance. We implement the four canonical rules as
proper diagnostic classifiers. We ship the result in two equivalent
surfaces, namely a standard Python library and an in-browser
frontend that runs the same wheel via Pyodide.

\section{Software description}
% =====================================================================

\subsection{Software architecture}

\hea{} is organized into a small set of orthogonal sub-packages and
a thin command-line interface. The \texttt{hea\_bench.composition}
module performs formula parsing and composition normalization, with
a single regular-expression-based parser that handles all three
source formula conventions, namely the space-separated proportional
amounts of \citet{borg2020}, the compact form of \citet{pei2020},
and the bare element strings of \citet{peivaste2023}. The
\texttt{hea\_bench.descriptors} module implements the six empirical
descriptors used by the rule set, which are $\Smix$, $\delta$,
$\mathrm{VEC}$, $\Tm$, $\Hmix$ \citep{miedema1973,miedema1980}, and
$\Omegaparam$ \citep{yang2012}. Per-element parameters live in the
elemental data table, which currently covers 24 elements. Binary
mixing-enthalpy pairs are loaded from a vendored copy of matminer's
Miedema pair table \citep{ward2018}, licensed under BSD-3-Clause
with the license preserved alongside the data file. This table
derives from the de Boer parameter set \citep{deboer1988} and is
consistent with the open tabulation of \citet{takeuchi2005}.

The \texttt{hea\_bench.rules} module contains the four canonical
empirical rules wrapped as classifiers, each in its own submodule
named \texttt{yeh\_smix}, \texttt{zhang\_delta}, \texttt{guo\_vec},
and \texttt{yang\_omega}. Each rule exposes a tunable threshold,
defaulting to the value reported in the original publication. The
\texttt{hea\_bench.classifiers.diagnostic\_stats} module provides
pure-stdlib implementations of Wilson score intervals
\citep{wilson1927}, confusion matrices, binary classifier
statistics, and receiver-operating-characteristic sweeps with
Youden's $J$ \citep{youden1950}. The \texttt{hea\_bench.benchmark}
module holds the per-source loaders, the cross-source consolidator,
and a coverage analysis that quantifies which fraction of the
consolidated dataset is scorable by the current elemental data
tables. The \texttt{hea\_bench.evaluate} module is the orchestrator
that runs each rule against the consolidated benchmark and writes
\texttt{rule\_baselines.json} alongside the consolidated CSV file.

Two end-user surfaces consume this library. The Python surface is a
standard \texttt{pip}-installable package, accessible via
\texttt{pip install hea-bench}. The browser surface is a single
HTML page that loads Pyodide \citep{pyodide} and invokes
\texttt{micropip.install} on the same wheel that
\texttt{pip} would deliver. Descriptor and rule calls happen
client-side in the user's browser tab, with no server-side
component. The library has one canonical implementation, and both
surfaces run it identically.

\subsection{Software functionalities}

The package supports six primary workflows. A user can parse an
arbitrary composition formula, normalize it to mole fractions, and
report the six standard descriptors. A user can apply each of the
four canonical empirical rules and obtain a predicted phase
classification. A user can score a user-supplied rule or
composition set against the consolidated v0.1.0 benchmark with
full diagnostic statistics. A user can sweep a rule threshold and
report the receiver-operating-characteristic curve, the
optimal-accuracy threshold, and the optimal Youden's-$J$ threshold.
A user can analyse element coverage of an alloy set against the
descriptor data tables and report next-best-add elements.
Finally, a user can re-consolidate the per-source data via the
bundled loaders and the deduplication and conflict-detection logic.

\subsection{The consolidated benchmark}

\hea{} v0.1.0 ships a single canonical benchmark at the path
\texttt{data/consolidated/v0.1.0/consolidated.csv}, containing
\num{7784} unique compositions. It is built by running the
per-source loaders on three open datasets. \citet{borg2020}
contributes \num{740} unique \texttt{(formula, processing)} alloys,
licensed CC-BY-4.0 and mirrored verbatim with a recorded SHA-256.
\citet{pei2020} contributes \num{1209} unique formulas, also
CC-BY-4.0 and mirrored. \citet{peivaste2023} contributes
\num{7747} unique formulas after deduplication of an
upstream-generated combinatorial sweep. The upstream data
repository for the \citet{peivaste2023} dataset declares no
license and is therefore pointer-only, fetched on user request at
build time.

Compositions are merged on a 4-decimal-precision composition key.
The processing column from \citet{borg2020} is preserved as a
side-channel rather than included in the join key. When two
sources assign different canonical labels to the same composition,
the row is flagged with \texttt{has\_conflict = 1} and the
per-source labels are preserved verbatim. The v0.1.0 release
contains \num{100} such cross-source conflicts, or
\SI{1.3}{\percent} of the benchmark. These conflicts are
themselves a useful artifact, because they enumerate compositions
on which the open high-entropy alloy literature does not agree.

The canonical phase taxonomy is deliberately coarse for v0.1.0,
namely $\{\mathrm{BCC},\, \mathrm{FCC},\, \mathrm{HCP},\,
\mathrm{multi\text{-}phase}\}$, so that all three sources map into
it without lossy guessing. Finer per-source labels, including the
microstructure column from \citet{borg2020} with more than thirty
sublabels and the twelve-category column from \citet{peivaste2023}
distinguishing intermetallic and amorphous phases, are preserved
as side-channel columns.

\section{Illustrative examples}
% =====================================================================

\subsection{The Cantor alloy, end to end}

The canonical equimolar CoCrFeMnNi alloy of \citet{cantor2004}
serves as the field's reference high-entropy alloy and as our
regression sanity check. Using \hea{}, this alloy is parsed by
\texttt{hea\_bench.parse\_formula} into a normalized mole-fraction
dictionary. The descriptor calls then return
$\Smix = \SI{13.381}{J\per(mol\cdot K)}$, which is exactly $R \ln 5$.
The size mismatch is $\delta = \SI{3.164}{\percent}$. The valence
electron concentration is $\mathrm{VEC} = 8.000$ exactly. The
melting temperature averaged by the rule of mixtures is
$\Tm = \SI{1801.2}{K}$. The Miedema multi-component mixing enthalpy
is $\Hmix = \SI{-4.16}{kJ\per mol}$. The Yang-Zhang parameter is
$\Omegaparam = 5.794$. All four canonical rules classify the
Cantor alloy as high-entropy class, single-phase, FCC, which is
consistent with the original experimental characterization
\citep{cantor2004}. Every value above is pinned in the regression
test suite.

\subsection{ROC recalibration of the Zhang $\delta$ rule}

Treating the rules as proper classifiers admits a question the
canonical literature does not address. Specifically, what threshold
would maximize Youden's $J$ on this benchmark? Sweeping the
\citet{zhang2008} $\delta$ threshold from \SI{1}{\percent} to
\SI{12}{\percent} in \SI{0.1}{\percent} steps yields the result in
Table~\ref{tab:roc-zhang}.

\begin{table}[h]
\centering

$J = 0.075$. The dataset-optimal $J$ is \num{0.113} at $\delta <
\SI{2.5}{\percent}$, approximately 2.6 times tighter.}
\label{tab:roc-zhang}
\begin{tabular}{lrrr}

\midrule

The canonical \SI{6.5}{\percent} threshold is heavily skewed toward
sensitivity at the expense of specificity. The balanced-optimal
threshold on the consolidated benchmark is approximately 2.6 times
tighter. Whether $J = 0.113$ constitutes a useful classifier is a
separate question, and it does not, but the recalibration finding
itself is a direct and defensible output of treating the rule as a
diagnostic classifier rather than a fixed cutoff.

\section{Impact}
% =====================================================================

\hea{} consolidates an open benchmark that previously did not exist
in unified form, and supplies the diagnostic-classifier machinery
that previously was not applied to the canonical high-entropy alloy
empirical rules in the open literature. We expect three classes of
impact.

The first is calibration of expectations. The collapse of both
binary rules to near-zero Youden's $J$ on the consolidated
benchmark is a quantitative caveat for downstream users of those
rules. The widely-quoted \SI{80}{\percent}-accuracy figures in the
original papers reflect different and smaller datasets and a
different choice of evaluation metric. On a consolidated open
dataset the rules behave very differently.

The second is the provision of a substrate for future methods work.
Any candidate model, CALPHAD-derived classifier, or new empirical
descriptor can now be scored against the same dataset and the same
diagnostic-statistics vocabulary as the four canonical rules. The
result is a head-to-head comparison rather than incommensurable
single-paper accuracies.

The third is accessibility. The Pyodide frontend lets any researcher
or student type a composition into a browser tab and see the same
numbers the published library would produce on a local install,
with the wheel itself loaded into the browser rather than a separate
JavaScript reimplementation. This removes the install-friction
barrier without sacrificing the library-as-canonical-implementation
guarantee.

\section{Conclusions}
% =====================================================================

This article has presented \hea{} 0.0.1, an open Python library and
benchmark suite for high-entropy alloy phase prediction. The
package ships a \num{7784}-composition consolidated benchmark drawn
from three primary open sources with explicit cross-source
provenance and conflict tracking. It provides reference-baseline
diagnostic-classifier evaluations of the four canonical empirical
rules. It reports a receiver-operating-characteristic recalibration
finding for the \citet{zhang2008} $\delta$ rule, showing a
dataset-optimal threshold of $\delta < \SI{2.5}{\percent}$ rather
than the canonical $\delta < \SI{6.5}{\percent}$. It includes an
in-browser Pyodide frontend that runs the same Python wheel a local
\texttt{pip install} would deliver. The package is MIT-licensed,
covered by 151 tests including pinned regression tests on every
headline benchmark number, and intended to serve as the substrate
on which subsequent methods for high-entropy alloy phase prediction
can be compared.

Planned future work includes expanding the elemental-property table
to lift benchmark coverage from \SI{86.7}{\percent} to approximately
\SI{95}{\percent}, since the additions are low-effort because the
corresponding pair-enthalpy entries are already present in the
vendored Miedema table. A v0.2.0 release will expose the
cross-source label conflicts as a structured side-dataset for
downstream disagreement-resolution work.
